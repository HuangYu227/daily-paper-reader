## 摘要
训练鲁棒的多元时间序列预测模型需要大规模、多样化的语料库，然而许多真实领域仅提供少量观测序列。现有生成方法无法解决这一矛盾：基于先验的方法（如CauKer、TimePFN）生成与领域无关的样本，而数据驱动方法（如TimeGAN）将参考序列视为黑盒监督，丧失了对于周期结构、局部变异和跨变量动态的显式控制。我们提出ReGeN，一种参考引导的生成流水线，它将观测序列不是作为模仿的示例，而是作为可控合成的结构支架。ReGeN将每个参考序列分解为三个可解释组件：捕获主导领域形态的相位对齐周期骨干；使用深度核高斯过程建模的每变量随机残差；以及通过带有拟合耦合系数的结构因果模型注入的滞后感知跨变量依赖关系。以可控温度对这些组件进行采样，可以拓宽分布覆盖范围，同时保持基于领域的结构。我们证明，ReGeN生成的数据能够持续替代真实兄弟数据且预测性能下降极小，并且在交通等强周期领域中，甚至可能优于真实源数据。我们还证明，在ReGeN语料库上预训练的基础模型优于在基于先验和数据驱动的合成替代方案上预训练的模型。这表明，在数据稀缺情况下，如何结构化利用参考数据可能与数据量本身同样重要。

## Abstract
Training robust multivariate time series forecasting models requires large, diverse corpora, yet many real-world domains provide only a handful of observed sequences. Existing generators fail to resolve this mismatch: prior-based approaches (e.g., CauKer, TimePFN) produce domain-agnostic samples, while data-driven methods (e.g., TimeGAN) treat references as black-box supervision, forfeiting explicit control over periodic structure, local variability, and cross-variable dynamics.   We propose ReGeN, a reference-guided generative pipeline that treats observed sequences not as examples to imitate, but as structural scaffolds for controllable synthesis. ReGeN decomposes each reference into three interpretable components: a phase-aligned periodic backbone capturing dominant domain morphology; per-variable stochastic residuals modeled with a deep-kernel Gaussian process; and lag-aware cross-variable dependencies injected through a structural causal model with fitted coupling coefficients. Sampling these components at controllable temperature broadens distributional coverage while preserving domain-grounded structure.   We show that ReGeN-generated data consistently substitutes for real sibling data with minimal forecasting degradation, and in strongly periodic domains such as traffic, can outperform the real source itself. We further show that a foundation model pretrained on ReGeN corpora outperforms those pretrained on prior-based and data-driven synthetic alternatives. This suggests that in low-data regimes, how reference data is structurally exploited can matter as much as how much data is available.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 总体框架
**ReGeN** 将每个参考序列（多变量，C个协变量，长度T）分解为三层结构：
- **相位对齐的周期性模板（Periodic Backbone）**：捕获主导的领域节奏形态。
- **每变量随机残差（Stochastic Residuals）**：使用深度核高斯过程建模局部不确定性。
- **跨变量依赖（Cross-variable Dependencies）**：通过拟合的结构因果模型（SCM）注入滞后依赖。

### 2.2 关键技术细节

#### (1) 结构分解（公式1-3）
对每个协变量c，在归一化空间中分解为：
\[ y_{c,t} = \tau_{c,t} + r_{c,t} \]
其中 \(\tau_{c,t}\) 是重复性结构模板，\(r_{c,t}\) 是残差。

**模板估计**：
- 使用特征周期P（数据集指定）对时间进行相位对齐，将同一相位的时间点取平均得到模板 \(\tau_c(p)\)。
- 残差为 \(r_{c,t} = y_{c,t} - \tau_c(t \mod P)\)。

**质量筛选**：
- 计算方差解释量 \(VE = 1 - Var(r)/Var(y)\)，仅保留VE高于阈值\(\eta\)的序列用于后续建模。
- 将这些高VE序列平均得到滤波后的均值信号 \(\bar{y}_{c,t}\)，再计算其模板 \(\bar{\tau}_c\) 和均值残差 \(\bar{r}_{c,t}\)。

#### (2) 模板结构支架（公式8）
生成时，对于选定的模板 \(\tau_{c,m}\)，结构贡献为：
\[ \hat{\tau}_{c,t} = \bar{\tau}_{c,m} + a_c (\tau_{c,m}(t \mod P) - \bar{\tau}_{c,m}) \]
其中 \(a_c\) 是每协变量振幅因子，保持形态但允许不同变量不同强度。

#### (3) 深度核自回归残差建模（公式9-11）
将均值残差 \(\bar{r}_{c,t}\) 建模为自回归过程：
- 构建上下文窗口 \(u_{c,t} = [\bar{r}_{c,t-W}, ..., \bar{r}_{c,t-1}]\)。
- 通过CNN+LSTM编码器映射到潜在表示 \(z_{c,t} = g_\theta(u_{c,t})\)。
- 使用深度核高斯过程：\(f_c \sim GP(0, k_c(z,z'))\)，残差 \( \bar{r}_{c,t} | u_{c,t} \sim N(f_c(z_{c,t}), \sigma_c^2)\)。
- 核函数结合Matern和RBF成分。

#### (4) 自回归残差生成与固有轨迹（公式12-13）
生成时：\(\hat{r}_{c,t} = \mu_{c,t} + \lambda_c \sigma_{c,t} \epsilon_t\)，其中\(\lambda_c\)是温度参数控制随机性。
加上一个小幅度正弦漂移项 \(d_{c,t}\)，固有轨迹为：
\[ \hat{x}^{int}_{c,t} = \hat{\tau}_{c,t} + \hat{r}_{c,t} + d_{c,t} \]

#### (5) 跨变量有向耦合（公式14-15）
基于从真实数据通过共识因果发现（PCMCI、DYNOTEARS、VARLiNGAM、随机森林Granger）估计的有向无环图G。对于子变量c，父节点贡献为：
\[ g_{c,t} = \sum_{p \in Pa(c)} \sum_{\ell \in L_{p \to c}} w_{p,c,\ell} s_{p,t-\ell} \]
最终子变量轨迹为：
\[ \hat{x}_{c,t} = \alpha_c \hat{x}^{int}_{c,t} + (1-\alpha_c) h(g_{c,t}) \]
其中 \(\alpha_c \in [0.7,0.9]\) 保证内生结构优先，\(h\) 可选的非线性变换。

## 3. 实验设计

### 3.1 数据集
使用12个真实数据集，覆盖5个领域：
- **能源**：BDG-2 Bear, Panther, Bull, Hog（来自BuildingsBench）

每个领域定义源-匹配兄弟数据集对（共享物理过程、传感器模态和时间粒度），用于跨域迁移测试。

### 3.2 评估协议
- **TRTR**（Train on Real, Test on Real）：真实数据训练和测试，作为上界。
- **TSTR**（Train on Synthetic, Test on Real）：合成数据训练，测试真实数据，孤立评估合成数据价值。
- **TRSTR**（Train on Real + Synthetic, Test on Real）：联合训练衡量增强收益。

所有设置下合成语料库大小与真实语料库大小匹配。

### 3.3 预测骨干网络
- **iTransformer**（~1M参数，自注意力机制）
- **DLinear**（4,673参数，显式季节-残差分解）
- **S-Mamba**（~1.1M参数，状态空间序列建模）
- **Moirai-small**（~13.8M参数，掩码编码器基础模型）

输入96步，预测24步，评估指标MSE和MAE。

### 3.4 对比方法
- **TimeGAN**（参考引导的对抗生成器）
- **CauKer**（无参考的基于先验的因果生成器）

在全语料库TSTR比较中，所有方法生成同等大小的语料库，Moirai-small从零预训练并在统一真实基准上零样本评估。

## 6. 论文的主要结论与发现

1. **参考引导合成数据可有效替代真实兄弟数据**：在72组TSTR设置中，67%（48/72）的MSE相对差异在±3%以内。在交通等强周期领域，合成数据甚至优于真实数据（PEMS场景降低6-10% MSE）。
2. **增强效果明显，尤其对注意力和状态空间模型**：TRSTR（真实+合成）在大多数数据集上匹配或击败TRTR，iTransformer和S-Mamba获益最大；DLinear由于自身显式分解，获益较小。
3. **优于现有生成器**：
   - 与TimeGAN相比，ReGeN在10/12数据集上更优（iTransformer），平均MSE降低41.1%（Moirai-small）。
   - 与CauKer相比，ReGeN预训练的Moirai-small MSE降低2.3%。
4. **组件重要性**：残差在低VE域（如Bull、Hog、Azure）中关键；SCM混合在多数多变量数据集上提供一致增益，但在低数据高维度（如Hog）可能有害。
5. **主结论**：在低数据场景下，如何结构化利用参考数据与数据量本身同等重要。

## 7. 优点

- **方法上的创新性**：将参考序列分解为可解释、可控的三层结构，区别于黑盒GAN和纯先验方法。
- **模块化设计**：每个组件可独立建模和消融，便于诊断和调整。
- **适应低数据场景**：不需要大量真实序列训练生成器，仅需少量参考序列即可抽取结构。
- **全面的实验验证**：12个数据集、3种协议、4种架构（含基础模型），消融实验支撑设计决策。
- **清晰的对比**：与TimeGAN和CauKer在完全匹配的设置下比较，证明参考引导策略的优越性。
- **分析深入**：通过VE、PSD、t-SNE多角度分析合成数据质量，并量化VE与残差重要性的关系（ρ ≈ -0.82，-0.89）。
- **实用价值**：表明在资源受限场景下，合成数据可充当真实数据的替代品，甚至提升模型性能。

## 8. 不足与局限

- **SCM混合不普适**：在低数据高维度（如Hog）中，因果图估计易受噪，导致性能轻微下降。作者建议未来使用自适应共识阈值。
- **计算约束**：全语料库预训练仅用Moirai-small，未验证其他基础模型（如Chronos、Lag-Llama）。消融仅针对iTransformer，未跨骨干验证组件影响的一致性。
- **特定弱域表现**：居民光伏（Residential PV）域，由于太阳辐照度局部变异大，模板拟合不佳，跨域迁移性能有限。

- **模板周期先验**：需人工指定特征周期P，对于非严格周期或复合周期数据可能不适用。
- **未与最新数据驱动生成方法比较**：如DiffTime、SSDSA等扩散模型未纳入对比。
