## 摘要
长上下文大语言模型（LLM）推理日益受到键值（KV）缓存的内存占用和解码成本的限制，导致在资源受限硬件上的可持续部署面临挑战。现有的KV缓存驱逐方法通常在基于GQA的LLM中对所有注意力头应用启发式令牌评分。这些方法忽略了注意力头的不同功能，导致关键令牌被驱逐，从而降低了LLM的性能。为解决这一问题，我们提出CompressKV，一种面向基于GQA的LLM的资源高效KV缓存压缩框架。CompressKV不聚合所有头的注意力分数，而是识别语义检索头（SRH），这些头能够捕获提示的初始和最终令牌以及语义重要的中间上下文证据，并利用它们选择应保留KV对的令牌。此外，CompressKV根据逐层驱逐误差的离线估计在各层之间分配缓存预算。在LongBench和Needle-in-a-Haystack上的实验表明，CompressKV在各种内存预算下均持续优于现有KV缓存驱逐方法。值得注意的是，在LongBench问答任务中，仅使用3%的KV缓存即可保留超过97%的全缓存性能；在Needle-in-a-Haystack中，仅使用0.7%的KV存储即可达到90%的准确率。这些结果展示了长上下文LLM推理中改进的资源-性能权衡。我们的代码公开于：https://github.com/TUDa-HWAI/CompressKV

## Abstract
Long-context large language model (LLM) inference is increasingly constrained by the memory footprint and decoding cost of key-value (KV) caches, limiting sustainable deployment on resource-constrained hardware. Existing KV cache eviction methods typically apply heuristic token scoring over all heads in GQA-based LLMs. These methods ignore the different functionalities of attention heads, leading to the eviction of critical tokens and thus degrading the performance of LLMs. To address this issue, we propose CompressKV, a resource-efficient KV-cache compression framework for GQA-based LLMs. Instead of aggregating attention scores from all heads, CompressKV identifies Semantic Retrieval Heads (SRHs) that capture both the initial and final tokens of a prompt and semantically important mid-context evidence, and uses them to select tokens whose KV pairs should be retained. Furthermore, CompressKV allocates cache budgets across layers according to offline estimates of layer-wise eviction error. Experiments on LongBench and Needle-in-a-Haystack show that CompressKV consistently outperforms existing KV-cache eviction methods across memory budgets. Notably, it preserves over 97\% of full-cache performance using only 3\% of the KV cache on LongBench question-answering tasks and achieves 90\% accuracy with just 0.7\% KV storage on Needle-in-a-Haystack. These results demonstrate an improved resource--performance trade-off for long-context LLM inference. Our code is publicly available at: https://github.com/TUDa-HWAI/CompressKV

# CompressKV: 语义检索引导的KV缓存压缩——详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：长上下文大语言模型（LLM）推理中，KV缓存的存储和解码计算成本随序列长度线性增长，成为资源受限硬件上的主要瓶颈。
- **现有方法局限**：主流KV缓存驱逐方法（如SnapKV、StreamingLLM）对所有注意力头统一评分，忽略了GQA（分组查询注意力）模型中不同注意力头的功能异质性。特别是“流式头”（Streaming Heads）倾向于只关注提示的开始和结尾，导致关键中间上下文token被错误驱逐，降低模型性能。
- **本文目标**：设计一种资源高效的KV缓存压缩框架，通过识别并利用“语义检索头”（Semantic Retrieval Heads, SRH）来指导token选择，并结合层间自适应预算分配，在极低缓存预算下保持高性能。

## 2. 方法论

### 核心思想
- 利用不同注意力头的功能差异：某些头（语义检索头）能够捕捉与语义相关的中间证据，而不仅是复制-粘贴行为。
- 基于这些头选择重要token，避免流式头主导的驱逐。
- 通过离线计算每层压缩误差，按需分配缓存预算。

### 关键技术细节

1. **语义检索头（SRH）识别**  
   - 使用标注数据（来自[27]的校准集）评估每个头对答案跨度（answer span）的注意力聚合。  
   - 定义SRH分数：  
     \[
     S_{\text{SRH}}(h) = \sum_{t=1}^N \mathbf{1}\{y_t \in \mathcal{A}\} \sum_{j \in \mathcal{A}} a^{(h)}_{t,j}
     \]  
     其中 \(y_t\) 是生成token，\(\mathcal{A}\) 是答案跨度，\(a^{(h)}_{t,j}\) 是头 \(h\) 在token \(t\) 对跨度内token \(j\) 的注意力权重。  
   - 相比传统检索头（依赖top-1命中），SRH通过聚合整个跨度的注意力，能识别出那些虽未精确匹配答案但贡献语义上下文的头。

2. **基于SRH的token选择**  
   - 每层选择得分最高的前 \(k\) 个SRH（实验中 \(k=4\)）。  
   - 对选中的SRH，将其在观察窗口内的注意力分数进行求和、池化、平均，得到每个token的重要性分数，保留分数最高的 \(N\) 个token的KV对。

3. **误差感知的层自适应缓存分配**  
   - 在离线阶段，对每个层 \(l\) 计算基于最小压缩（极端压缩）的Frobenius范数误差：  
     \[
     e^{(l)} = \sum_{t=1}^T \frac{\|\Delta O_t^{(l)}\|_F}{\|O_{\text{full},t}^{(l)}\|_F + \epsilon}
     \]  
     其中 \(O_{\text{full}}, O_{\text{comp}}\) 分别为全缓存和压缩缓存的注意力块输出。  
   - 归一化误差 \(\tilde{e}^{(l)}\) 后，在总预算 \(B_{\text{total}}\) 下，为每层分配最小预算 \(m=32\) 和最大预算 \(M=3 \times B_{\text{per-layer}}\)，剩余预算按误差比例分配。  
   - 该分配过程离线完成，在线推理无额外开销。

## 6. 主要结论与发现

- CompressKV在所有KV预算和模型上一致优于基线方法，尤其在极低预算下优势显著。
- **LongBench**：仅用3%缓存（平均每层256 tokens）即可保留97%以上全缓存性能；用19%缓存（1024 tokens）保留超过99%。
- **NIAH**：Llama-3.1-8B上，0.7%缓存（256 tokens）达到90%准确率；2048 tokens（5%缓存）接近无损。
- SRH比传统检索头（TRH）更有效：mask SRH导致NIAH性能大幅下降（Top-30时下降73.81%），而mask TRH仅下降13.30%。
- 层自适应分配带来额外增益，且与SRH选择互补。
- CompressKV可正交集成于预填充加速、量化、头级分配等现有技术，进一步提升效果。
