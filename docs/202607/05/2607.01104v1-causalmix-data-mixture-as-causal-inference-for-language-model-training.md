## 摘要
在大语言模型（LLM）训练中，数据混合对模型性能起着关键作用。近期方法通过代理模型优化混合权重，但依赖于静态数据分布的假设。因此，当底层数据池发生变化时，这些方法需要从头开始进行昂贵的重新训练。这一局限性限制了它们从小规模设置无缝扩展到更大数据池和模型规模的能力。在本文中，我们提出CausalMix，通过将数据混合优化建模为因果推断问题来解决这一局限性。我们将数据池的统计特征作为协变量，领域混合作为处理变量。在基于512次Qwen2.5-0.5B运行拟合因果模型以估计条件平均处理效应（CATE）后，我们外推了适用于800K数据池的最优混合，并将其用于训练7B模型。此外，我们成功将该框架泛化到Qwen3-4B-Base上的长思维链数据。通过利用因果建模隔离混杂偏差，CausalMix动态推断状态依赖的最优数据混合。大量实验表明，由CausalMix指导的混合在多个下游任务上持续提升性能，优于RegMix及其他基线。此外，我们使用CATE解释器对学习到的混合策略进行可视化分析。总体而言，CausalMix为优化LLM数据混合提供了一个因果且可解释的框架。

## Abstract
In Large Language Model (LLM) training, data mixing plays a pivotal role in determining model performance. Recent methods optimize mixture weights via proxy models, but they rely on the assumption of static data distributions. As a result, when the underlying data pool shifts, these methods require costly retraining from scratch. This limitation restricts their ability to scale seamlessly from small settings to larger data pools and model sizes. In this paper, we propose CausalMix to address this limitation by casting data mixture optimization as a causal inference problem. We formulate the statistical features of the data pool as covariates and the domain mixture as the treatment. After fitting a causal model on 512 runs of Qwen2.5-0.5B to estimate the Conditional Average Treatment Effect (CATE), we extrapolate the optimal mixture for an 800K data pool and apply it to train a 7B model. Furthermore, we successfully generalize the framework to long chain-of-thought data on Qwen3-4B-Base. By leveraging causal modeling to isolate confounding biases, CausalMix dynamically infers state-dependent optimal data mixtures. Extensive experiments show that the mixture guided by CausalMix consistently improves performance across multiple downstream tasks, outperforming RegMix and other baselines. In addition, we use the CATE Interpreter to provide visual analysis of the learned mixing strategy. Overall, CausalMix offers a causal and interpretable framework for optimizing LLM data mixtures.

- **整体含义**：本文提出 CausalMix，将数据混合优化从黑盒超参数搜索重新框架为**因果边际回报估计**问题。通过引入数据状态（data state）作为协变量，CausalMix 能够动态推断状态依赖的最优混合，并具有可迁移性和可解释性。

### 2. 论文提出的方法论

- **核心思想**：不再直接学习混合比例 → 性能的全局映射，而是问一个局部的因果问题：“在当前数据状态下，相对改变某个领域的比例，对下游性能的因果影响是什么？”
- **关键技术细节**：
  - **定义**：将数据统计特征（如 HES、Normalized_Loss、Writing_Style）作为协变量 **X**，域混合比例 T（经对数变换 Z = log(T + ε)）作为连续处理变量，下游任务的微平均得分作为结果 **Y**。
  - **模型**：采用部分线性近似 μ(x, z) ≈ g(x) + θ₀(x)ᵀ Z，其中 θ₀(x) 是状态条件边际数据回报（即 CATE 的推广）。
  - **估计**：使用双机器学习（Double Machine Learning, DML）进行正交化估计：先用 LightGBM 拟合残差 Ỹ = Y - E[Y|X] 和 Ẑ = Z - E[Z|X]，再通过 R-loss 目标学习 θ(x)。
  - **策略提取**：两种方式：
    - **CausalMix-A（解析法）**：对正边际回报做 ReLU + L1 归一化。
    - **CausalMix-S（搜索法）**：从 Dirichlet 分布中采样大量候选混合，用因果模型评分并平均 top-100 结果。
- **算法流程**：分为三个阶段：
  1. 历史代理训练：用不同混合比训练 512 个 Qwen2.5-0.5B 模型，收集 (X, T, Y) 三元组。
  2. 因果模型拟合：用 Cross-fitting 和 CausalForestDML 估计 CATE。
  3. 策略外推：将目标数据池的协变量 X 输入模型，通过 A 或 S 方式得到最优混合，直接用于训练更大模型（7B）或不同数据池（AM-Thinking）。

### 6. 论文的主要结论与发现

- **性能优势**：CausalMix（尤其 CausalMix-A）在大多数设置下取得最高 Avg Dev 分数，在 7B 规模上相比 DMO、RegMix 有显著提升。
- **可迁移性**：在 tulu-3 上训练的因果模型可直接应用于 AM-Thinking（不同数据池、不同模型系列 Qwen3-4B），无需重新训练代理模型，且在 Math 和 Code 能力上取得最佳平均分。
- **可解释性**：通过 CATE 决策树分析发现：
  - IF 数据始终正面贡献；
  - 在高质量（高 Writing_Style、高 HES）的数学、编程数据区域，Safety 数据也能产生协同增益；在低质量区域则有害；
  - Knowledge 数据在高 Normalized_Loss 和高 HES 区域呈现负效应，证实“技能冲突”（逻辑推理 vs 事实知识）。
- **消融实验**：去除 DML 正交化或去除协变量都会导致性能下降，验证了因果框架和状态条件化的必要性。

### 7. 优点

1. **方法论创新**：首次将因果推断（DML + CausalForest）系统引入 LLM 数据混合优化，从静态回归升级为动态因果边际估计。
2. **可迁移性**：学习底层因果动态而非记忆特定数据集，能零外推到未见数据池和更大模型。
3. **可解释性**：CATE 解释树明确揭示了不同域在不同数据状态下的边际贡献，为调试和策略设计提供依据。
4. **模块化设计**：代理模型、因果估计、策略提取可独立改进，框架开放（基于 EconML、LightGBM）。
5. **实验设计扎实**：从 0.5B 到 7B 的缩放、不同数据量、消融和因果变量选择分析，逻辑完整。

### 8. 不足与局限

1. **因果识别假设的强依赖**：假设可忽略性（Y(t) ⊥ T | X），要求混合机制预先指定且不依赖训练反馈；实验中混合是随机采样的，尚可接受，但若未来使用自适应混合，该假设可能被违反。
2. **协变量维度受限**：由于只有 512 个代理样本，协变量仅选 3 个（HES, Normalized_Loss, Writing_Style）；论文指出更多协变量可能因维度灾难而下降，但若能增加代理数量或引入更好的降维，可能进一步提升性能。
3. **泛化实验覆盖有限**：扩展实验仅在一个 LongCoT 数据池（AM-Thinking）和一个模型（Qwen3-4B）上实施，更多 No-CoT、多语言、多任务场景有待验证。
4. **下游任务侧重 SFT**：方法目前只验证了 SFT 阶段，对于预训练阶段的行为未讨论（虽提及 RegMix 等）。
5. **计算成本**：虽然迁移时无需重跑代理，但初始的 512 个 0.5B 训练（~5.53e20 FLOPs）仍是一笔可接受但不可忽略的成本；对资源有限团队有一定门槛。
6. **统计显著性**：未提供置信区间或多次重复的标准差，尽管在多个规模下趋势一致，仍需更严格的统计验证。
